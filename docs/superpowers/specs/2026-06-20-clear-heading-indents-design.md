# 清除标题直接缩进设计

## 背景

激进路径只配置标题样式，不再逐段直接格式化。真实输出稿显示一至三级标题的有效首行缩进仍为 24 pt，而参考文献标题为 0。原因是原文标题段落保留了直接段落缩进，该直接格式优先于标题样式中的零缩进。

## 目标

- 题目以及大纲级别 1–3 的标题段落强制无首行、左侧和右侧缩进。
- 不修改标题字体、字号、对齐、行距、段前段后或分页设置。
- 不修改正文、摘要、目录条目、参考文献条目、图题和表格。
- 保持激进路径的主要性能收益。

## 实现

新增 `ClearHeadingParagraphIndents`：使用一次 `For Each` 遍历文档段落，仅识别题目样式或大纲级别 1–3 的段落，并将：

```vb
.FirstLineIndent = 0
.LeftIndent = 0
.RightIndent = 0
```

主流程在 `ConfigureSDUTCMStyles` 后调用该过程，使样式先完成配置，再清除原文标题上的直接缩进。

参考文献标题仍由现有参考文献流程处理；重复设置为 0 是幂等行为。

## 验证

- 静态测试确认主流程调用标题缩进清理过程。
- 静态测试确认该过程只写入三种缩进属性，并限制在题目和大纲级别 1–3。
- 使用真实文档临时副本验证所有大纲级别 1–3 标题的有效缩进均为 0。
- 运行现有三项回归测试和完整性能测试。
