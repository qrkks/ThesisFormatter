# 字体格式化性能修复设计

## 背景

当前 `ApplyRangeFontPreservingItalic` 会遍历段落中的每个字符，逐个读取斜体和颜色，再逐个设置字体属性。正文和参考文献都会调用该过程，导致运行时间随全文字符数和 Word 对象调用次数急剧增长。

## 目标

- 将正文与参考文献的字体设置恢复为整段批量操作。
- 统一字体名称、字号和粗体。
- 保持原有斜体和颜色不变。
- 防止以后重新引入逐字符字体循环。

## 非目标

- 不改变段落缩进、行距或对齐方式。
- 不调整标题、目录、分页、页码、图片或表格逻辑。
- 不取消直接格式化兜底，也不改为完全依赖 Word 样式。

## 设计

保留现有字体辅助过程及调用点，避免扩大修改范围。辅助过程改为直接操作传入 Range 的 Font：仅设置 `NameFarEast`、`Name`、`Size` 和 `Bold`。不读取或写入 `Italic` 与 `Color`，因此这两个属性自然保持原样。

移除 `contentRange.Characters` 遍历及字符级临时状态。正文和参考文献继续通过同一个辅助过程应用规则，行为保持一致。

## 测试

先修改静态回归测试并确认其在旧实现上失败。测试要求：

- 辅助过程不得遍历 `Characters`。
- 辅助过程必须整段设置字体名称、字号和粗体。
- 辅助过程不得读取或设置 `Italic`、`Color`。
- 正文和参考文献仍调用该辅助过程。

实现后运行全部现有 PowerShell 回归测试，并执行 `git diff --check`。由于仓库没有可直接运行 VBA 的自动化测试框架，Word 性能微基准作为诊断证据保留，不纳入常规测试。
