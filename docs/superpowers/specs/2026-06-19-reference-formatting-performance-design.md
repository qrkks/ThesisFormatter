# 参考文献格式化性能优化设计

## 背景

在 35 页、511 段的实际文档上，参考文献阶段耗时 94.3 秒，是当前最慢的单个阶段。现有主流程依次调用标题规范化、标题格式化和条目格式化；后两步使用 `ActiveDocument.Paragraphs(i)` 再次扫描全文，并对每个参考文献条目逐段设置格式。

## 目标

- 主流程只定位一次参考文献标题。
- 从标题后开始寻找参考文献结束边界，不再重复扫描标题之前的内容。
- 对参考文献条目区间进行一次批量格式化。
- 保持当前标题、悬挂缩进、行距、字体、分页规则不变。
- 不设置斜体和颜色，使已有斜体、颜色自然保留。
- 在指定实际文档上将参考文献阶段从 94.3 秒降到 15 秒以内。

## 非目标

- 不修改正文、摘要、目录、表格、图片或页码流程。
- 不启用参考文献自动排序或自动编号。
- 不删除现有可单独调用的参考文献宏，以免破坏外部使用方式。

## 方案

新增一个供完整流程调用的参考文献区间格式化过程。它按以下顺序工作：

1. 使用 `For Each` 从文档开头定位第一个参考文献标题。
2. 如果标题和第一条文献位于同一段，沿用现有规则拆成标题段和条目段。
3. 立即格式化标题段。
4. 从标题后的段落开始使用 `For Each` 查找结束边界。结束条件沿用当前规则：附录、图表、致谢、作者简介，或一至三级标题。
5. 如存在后续章节，为其设置 `PageBreakBefore`。
6. 建立从标题末尾到结束边界起点的单个 `Range`，一次设置段落对齐、悬挂缩进、行距、字体名称、字号和粗体。

完整流程 `ProcessReferences` 与 `ProcessReferencesWithSort` 改为调用该新过程。旧的 `FormatReferences`、`FormatReferenceEntries` 和其他公开入口保留，避免兼容性破坏，但主流程不再串行调用它们。

## 边界处理

- 找不到参考文献标题时直接返回，不修改文档。
- 标题后没有条目时只格式化标题。
- 没有后续章节时，条目区间延伸到文档末尾。
- 空段落包含在批量范围内；这只改变其潜在段落格式，不改变文本或结构。
- 批量字体设置不包含 `Italic` 和 `Color`。

## 测试与验证

- 先更新静态回归测试，要求完整流程调用区间化过程，并确认旧实现测试失败。
- 测试区间化过程使用 `For Each`，不存在 `Paragraphs(i)` 循环。
- 测试条目格式通过单个 `Range` 设置，且不写入 `Italic`、`Color`。
- 运行现有参考文献与表格回归测试。
- 在临时副本中运行当前完整宏，对参考文献阶段重新计时；验收阈值为 15 秒。
- 原始论文文档始终只读，性能测试只操作临时副本。
